# Roadmap

## Phase 1: Foundation
- Set up server-side TypeScript project structure.
- Add Express and SQLite support.
- Create a basic API for agents and appointments.

## Phase 2: Agent and appointment models
- Define agent data, ailments, therapies, and booking entities.
- Build CRUD endpoints for agents and appointments.
- Add simple validation and error handling.

## Phase 3: Dashboard and UI prototype
- Add a dashboard route for staff and agents.
- Implement a browser-friendly frontend shell.
- Surface agent relief status and upcoming appointments.

## Phase 4: Stakeholder polish
- Improve UI attractiveness and browser compatibility.
- Add dashboard filters for therapies and ailment status.
- Ensure reliability and ease of access for staff.
